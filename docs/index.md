# docs/

Documentation root. The agent entry file is `/AGENTS.md` at the repo root;
this folder holds the deeper material it links into.

| Folder / file | Purpose |
|---|---|
| [`reference/`](reference/) | Information-oriented: CLI surface, env vars, schemas. Look here to confirm a fact. |
| [`adr/`](adr/) | Append-only architecture decisions. Look here for *why* the system is shaped this way. |
| [`DATEV-DEVELOPER-PORTAL.md`](DATEV-DEVELOPER-PORTAL.md) | Hand-curated map of the DATEV developer portal and known API products. |
| [`SECURITY.md`](SECURITY.md) | Known security limitations (e.g. magic-link tokens in transport logs) and accepted tradeoffs. |
| [`README.md`](README.md) | Notes on the DATEV docs pack and how to refresh it. |

Tutorials, how-to guides, and explanation pages are intentionally absent —
they will be added when a human reader actually exists for one.
